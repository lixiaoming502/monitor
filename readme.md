## 概览
简单的跟踪程序，用来分析模块调用，找出调用时间长的模块。

## 接口

## 1. 事件汇报
### 1.1 功能描述

在事件的开始，结束，向此平台进行汇报

### 1.2 请求说明
> 请求方式：GET<br>
请求URL ：[/trace/reportme](#)

### 1.3 请求参数

字段       |字段类型       |字段说明
------------|-----------|-----------
module_name       |string        |模块名称
event_name       |string        |事件内容
trace_key       |string        |跟踪标示，如用户id
stage       |int        |事件阶段 0：开始 1：完成 2：进入事件

### 1.4 返回结果
正常情况下返回SUCCESS
